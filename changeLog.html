ChangeLog

May 26, 2024

  • Modified query results to go back to showing greater than the last target. I didn't want to do this but because damage-related queries in particular cast such a huge net, it is possible to have millions of duplicate equivalent results. Duplicate results found by multiple threads will still show, but duplicate results found within each thread will no longer be shared to the main thread that handles it all. TLDR: Less dulicate results will be shown now, in order to speed things up.
  • Fixed an issue that could cause queries to crash in scenarios where the user filtered so strictly the queries couldn't return enough rings to complete a loadout, resulting in the concoction limit boost sum of a ring set to be read as a ring itself.

May 25, 2024 - SPEED UPDATE #2

  • Reworked entirely how the conditional hierarchy works. This is the stuff that lets something like Generating Band see if there is a shield active in order to determine whether to give healing or not. This is purely a back-end change and you guys shouldn't see anything different other than queries are about 38% faster than they were yesterday.
    • As with all major reworks, I may have missed something jank. If you see anything that doesn't make sense that might have slipped past my testing, let me know.
  • Fixed an issue where if the user arrived at the site without interacting with literally anything and goes straight to the queries and pops in a target and hits start without putting in any filters, it could crash the calculator.
  • Fixed an issue where Alchemy Stone's lifesteal was provided regardless of whether the player was suffering from a status effect(like Atonement Fold's bleed effect). Not a huge deal since it still displayed the bonus correctly, just not always in the right context.
  • Resonating Heart's overheal effect UI element will now only display when the relic is not disabled. When disabled before, the numbers in overall calculations would adjust as they should, but the overheal UI element didn't have any resets in place and would linger when it wasn't relevant to the display.

May 18, 2024

  • Changed Blood Bond's trait bonus to only populate when Summoner is a selected archetype. We had discussed doing this before in light of the recent game change, but I didn't have anything set up for traits to allow custom conditionals yet, I added that today.
  • Added custom function call handling for traits. Right now Blood Bond is the only one that uses it(All other item types already use this), but it'll be nice to have for the future since traits were designed to be static bonuses when I started this and that may not always hold true.
  • Fixed an issue where Resonating Heart's custom UI elements for the overheal effect did not display in some circumstances.
  • Fixed an issue where Monolith's Sandstorm mod was giving extra stats it should not be.
  • Fixed an issue where Sandstorm's EXPOSED effect could apply twice within the UI.
  • Fixed an issue where the basic statistics column(armor, stamina, weight, etc) took up too much space on mobile. Simply forgot to set the height to auto on mobile devices and it still tried to take up desktop height, my b.

May 16, 2024

  • Added movement speed as a query target, and stats filter preset.
  • Added more lifesteal target statistics to queries so that way you aren't stuck with lifesteal peak. Peak assumed you didn't care what kind of lifesteal it was, and only wanted to see the highest lifesteal possible, but now you can target what type you want (all, ranged, melee, charged melee, etc)
  • Added handling for "Rusty" prefix weapons within the main UI and within cycles calculations. Rusty weapons cannot use a mod or mutator, this is specific to Rusty Lever Action primary, and Rusty Repeater secondary.
  • Added a heads-up display to the Damage Section to notify users of supported Damage Calculations. Later I'll change this to a hover-tooltip when we have more supported types.
  • Fixed an issue where if you locked a blank slot into the locks filters of a given category via entering "--" as the locks filters section states you can, it would break when the lock was iterated upon. Now you can lock as many slots as you want, with an empty item that neither has stats or uses. This may sound weird, but there are definitely uses for this, but only on locks.
  • Slight back-end code cleaning. Changes nothing performance related.

May 15, 2024

  • Added images to all mutators, they'll now display in the formerly empty box in a weapon's display tab.
  • Query results now store the top 50 builds, instead of the top 3. This was added because with the change I brought yesterday I forgot to mention that repeat value results will now show where they didn't before. I don't mean repeat builds, I mean repeat target values, so if two entirely different builds result in exactly 800 EHP on an EHP query, they'll both show up now, and stuff like that happens WAY more than you think it would lol. Before, if an equivalent target value was found, it was skipped because it wasn't greater than the target value already found.
  • Fixed an issue where if a query was forcibly aborted due to excessive possible combos, it didn't fully reset some UI elements necessary to modify or restart further queries. The user could still get around it by clicking "abort" as the button is attached to code that would reset it properly, but now it's fixed anyways.
  • Changed Bottom Feeder's name to Spirit Feeder like in game

May 14, 2024 -

THE QUERY SPEED UPDATE (Up to 90% faster)

  • Queries now offer an option to increase the number of threads on your CPU that queries can operate on. This will DRASTICALLY increase query speeds, but be sure to monitor CPU and RAM usage to make sure you didn't open too many, as while it does speed things up it also comes at a cost. Refer to your CPU's Core Count as a general rule of thumb.
  • Given that queries can now be much faster, I've bumped the query length limit from 300 Milllion, to 10 Billion.
  • Fixed an issue where if a user duplicated a browser tab with a target statistic selected in query settings, sometimes the target query statistic would copy over but wouldn't actually apply, resulting in the user needing to delete it and add it back before it would register again.
  • Fixed a typo in Twisted Idol's description.
  • Updated Dark Pact's bonuses, it still reflected the preDLC2 bonuses.
  • Updated a few straggler Fragment bonuses I missed, like Ranged Crit Chance (10% -> 7.5%)
  • Blood bond no longer works on anything other than summoner minions, in-game. The calculator does not reflect this in terms of calculations, but I did update Blood Bond's description to make that explicitly clear.
  • Added tooltips to the following. Forgot what a given trait did, or even what its bonus was at a given level? Wonder no longer, they will show!
    • Traits
    • Fragments
  • Added regex color filtering for GLOOM, BRITTLE, and DRENCHED from invoker abilities. Added blank(white) filtering for EXPOSED and GIFT OF THE FOREST.

May 13, 2024

  • Queries in progress will now only update the counter progress bar if either a new best build is found, or if the current iteration is an increment of 10,000. This helps reduce communication overhead between the web worker and the main thread, increasing query speeds by up to 25% in some cases. THAT is def gonna help on speeds, but I understand some people won't feel like it's faster since the numbers won't flick by anymore. Also note it is not guaranteed to make things any faster depending on the type of query initiated. Calculation-intensive queries might still take a little bit longer.

    INTERFACE REWORK

  • Added regex filtering to include numbers within descriptions. No matter the units used(seconds, meters, %, etc) they will now be emboldened like the game UI.

    • CALCULATIONS & ADJUSTMENTS

  • Modified all item statistics and custom functions as best I could, to match pre-release patch notes. I say best I could as the patchnotes were not incredibly specific on every change that happened, some mention value changes but not how much. As such, many values remain unchained until we can get the DLC update to see what the new values are.
  • HASTE is now calculated and its bonuses displayed where relevant
  • Added support for up to 4 quick-use consumables. Before I only supported 2 because I firmly believed there were never more than 2 that related to tanking, but now that we're moving beyond tanking I obviously need to change that. Rejoice, anyone who imported builds with more than 4 quick-use from R2TK.
  • Added conditional calculations for over 40 items. If I was able to make it conditional without being too excessive, then I did it. Stuff like Alchemy Stone or even Embrace of Sha'hala no longer provide their bonuses implicitly. I tried to specify within item descriptions if the condition was assumed to always be true, so if you don't see that then you can generally assume that there is a conditional associated, but I might have forgotten a few so let me know.
  • Added outgoing/incoming status effects, based on what your build is, for the sake of conditional calculations mentioned above. An example: Atonement Fold provides BLEEDING as an incoming status to the player. The player now counts as an entity afflicted with both a negative STATUS effect, and a BLEEDING status effect.
  • Added advanced stats settings for: Enemy Count, Grey Health Active, Perfect Dodge, Evade dodge override(otherwise it assumes it is a normal dodge/roll, not an evade/backstep)
  • Added the distinction between stamina cost and action cost, as not all stamina cost reductions are the same thing
      So as an example:
    • Evades cost 25 stamina at base. Say you have 20% reduced evade cost, this is an action cost reduction, and applies to the base cost of the action BEFORE stamina cost penalties or bonuses are applied.
      • This logic applies to all stamina-related actions and their associated action-cost specific stats.
  • Removed stats from Samoflange. It used to only show the negative DR gained in the downtime, but I agree that isn't fair to do since it does have a positive. However, even if we average the DR gained over time using the bonus and the negative, you're still a hair under 0% DR gained and negative. As such, it is simplest to just stop the item from adding anything anymore. Why in the hell would you be using that thing anyways.
  • Removed the ability to toggle calculations for relic fragments, and traits. These were mainly there for when this was still a spreadsheet so you could verify the source was providing what it should be, but for fragments all you need to do to do that is remove the fragment, and for traits just unrank them. I may add this back later if I find a good way to incorporate it into the UI that isn't pointless, but for now they go.
  • Removed Wayfarer and Resonance, and merged them with Swiftness and Amplitude, respecitvely. Added support for multi-benefit, separately-scaling traits in the calculations.
  • Adjusted stamina penalty calculations for Verdant Tea. It doesn't list it as such within the game, but it does reduce the weight-cost penalty. Upddated the description on my end to reflect that.
  • Made slight changes to the overall save structure of the URL parameters. If you notice anything weird let me know immediately. It will probably screw with people who had a complex relic input saved in the URL for Resonating Heart, as that setting is now fully calculated without user input and the save slot has been replaced by enemyCount

  • Fixed an issue where Game Master's Pride was not calculating shared damage correctly. I'm amazed nobody noticed before frankly, but that was a mistake on my part.
  • Fixed an issue where SHIELD EHP did not properly calculate when Restriction Cord was selected. This was because Restriction Cord does not actually cut your HP in half for the purposes of shielding, it only prevents you from healing past half.
